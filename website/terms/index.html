You may use the software to connect compatible MCP clients to your own Google account and perform Google Docs and Drive actions through the available MCP tools.

You are responsible for the prompts, tool calls, document IDs, sharing targets, OAuth scopes, and actions you authorize.

Google account and API access

The software uses Google OAuth. By authenticating, you authorize the app to use the Google scopes shown on the consent screen.

The default public profile requests Google Docs access and per-file Drive access through drive.file. Optional full-Drive mode is available only when you explicitly use the --full-drive profile.

The default architecture is local stdio. In that mode, the server runs on your machine and stores your Google OAuth token locally.
