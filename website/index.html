Purpose and Google data use

Built for user-directed document automation

Create and edit Docs

The server uses Google Docs access to create documents, read document content, insert text, format paragraphs, manage tables, headers, footers, links, comments, images, named ranges, and tabs.

Manage permitted Drive files

The default profile uses per-file Drive access so the server can rename, move, export, share, trash, restore, and inspect files created or explicitly available to the app.

Run locally by default

In the default stdio setup, the publisher does not receive your OAuth token or document content. Tokens are stored on your own computer.

OAuth access

Default mode is narrow; full Drive is opt-in

The public default profile requests Google Docs access and drive.file. This supports creating and continuing work on app-created documents while avoiding unrestricted access to every file in a user's Drive.

Users who intentionally need broad discovery of existing Drive files can use the optional full-Drive profile, which requests Google's restricted drive scope.

Default scopes https://www.googleapis.com/auth/documents https://www.googleapis.com/auth/drive.file
Optional full-Drive scope https://www.googleapis.com/auth/drive
